# OpenVINO Windows LLM 0.4.0

## Summary

This release establishes the Windows installer, portable distribution, first-run onboarding, system-tray lifecycle, privacy-safe diagnostics, and a production-oriented release pipeline for the local OpenVINO server.

## New features

- Per-user Windows installer and explicit portable mode.
- Hardware-aware first-run preparation and tray-owned server lifecycle.
- Optional stable and beta update discovery through official GitHub release manifests.

## Reliability fixes

- Single-instance ownership, bounded readiness, graceful shutdown, crash detection, and writable-data preflight.
- Persistent data-schema marker and safe refusal when a newer incompatible schema is detected.

## Security changes

- Release output scanning excludes secrets, certificates, model directories, caches, and unsafe ZIP paths.
- Signed state is reported only after Authenticode verification succeeds.
- Update checks transmit no hardware, model, prompt, chat, user, token, or analytics data.

## API compatibility changes

Chat Completions, Responses, embeddings, streaming, tool calls, and model lifecycle contracts are intended to remain compatible. Validate with the mock contract suite before publication.

## Model or device behavior changes

No new CPU, GPU, NPU, model, driver, or precision compatibility claim is made by these release-system changes.

## Packaging changes

- Canonical application version in `app/version.py`.
- Deterministic installer, portable ZIP, checksum, manifest, release-note, and third-party-license filenames.
- Isolated pinned release environment with resolved dependency inventory.
- Optional signing and mandatory post-sign verification for signed releases.

## Upgrade notes

Installed mutable data remains under `%LOCALAPPDATA%\OpenVINOWindowsLLM`. Portable mutable data remains under the extracted directory's `data` folder. Review `docs/UPGRADE_ROLLBACK.md` before downgrading.

## Known issues

Real Windows installer upgrade, rollback, Authenticode, CPU, GPU, and NPU paths remain unverified until the corresponding Windows procedures are executed and retained.

## Hardware validation performed

None recorded by this source implementation.

## Mock validation performed

Source-level unit validation is expected. Packaged mock validation must be executed with `-MockSmokeTest` on Windows before release publication.

## Unverified hardware paths

- Real Windows CPU inference
- Real Intel GPU inference
- Real Intel NPU inference
- Driver-specific behavior
- Installer upgrade and downgrade on an existing user profile
