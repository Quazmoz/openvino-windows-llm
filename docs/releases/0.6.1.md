# OpenVINO Windows LLM 0.6.1

## Summary

This patch release makes model-library certification deterministic and expands retained,
report-backed CPU and GPU coverage to Qwen 2.5 3B FP16 and TinyLlama 1.1B INT4.

## Reliability fixes

- Conversion cancellation terminates the complete Windows converter process tree so a retry
  does not inherit an orphaned Optimum exporter.
- The Windows harness validates exact prompt-depth boundaries and rejects the first prompt
  depth beyond the configured maximum.
- Compiled-cache reuse is established by stable cache metadata across restarted processes,
  without timing or throughput claims.
- AUTO results remain separate and cannot create a direct-device certification claim.

## Model library

- Adds TinyLlama 1.1B Chat INT4 as a practical low-footprint starter model.
- Adds retained CPU and GPU certification records for TinyLlama INT4 and Qwen 2.5 3B FP16.
- CPU certification records honestly use a null driver version; every record links to a
  retained report and its SHA-256.
- NPU and AUTO results from this cycle remain unverified in the bundled catalog because the
  NPU runs failed and AUTO did not expose an actual direct device.

## Packaging

Model weights and compiled caches remain excluded from the installer and portable ZIP. The
catalog, harness improvements, and small sanitized evidence bundle are the only package
footprint changes. Release signing status is reported by the build and publication tooling;
this note makes no signing claim.
