# OpenVINO Windows LLM 0.5.0

## Summary

This release carries forward the 0.4.0 Windows installer, portable distribution, first-run onboarding, tray-owned server lifecycle, privacy-safe diagnostics, and release pipeline, and fixes a system-tray menu defect that made hovering over menu items unreliable.

## New features

- No new end-user features. This is a stability release built on the 0.4.0 baseline.

## Reliability fixes

- The tray menu no longer glitches while hovering over items or opening submenus. The background status poller previously rebuilt the native menu handle on a timer, which could destroy the menu Windows was actively drawing; menu rebuilds are now deferred until the menu closes and serialized so two threads never destroy the same handle.
- Tray refreshes are coalesced so the icon, tooltip, and menu are only rebuilt when the visible state actually changes, eliminating the periodic rebuild churn behind the hover issue.

## Security changes

- No security-relevant changes since 0.4.0. Release output scanning, post-sign verification, and privacy-preserving update checks are unchanged.

## API compatibility changes

Chat Completions, Responses, embeddings, streaming, tool calls, and model lifecycle contracts are unchanged from 0.4.0 and validated with the mock contract suite before publication.

## Model or device behavior changes

No new CPU, GPU, NPU, model, driver, or precision compatibility claim is made by this release.

## Packaging changes

- Canonical application version advanced to 0.5.0 in `app/version.py`; installer, portable ZIP, checksum, manifest, release-note, and third-party-license filenames follow the deterministic 0.5.0 naming.
- Built from the isolated pinned release environment with a resolved dependency inventory.

## Upgrade notes

Installed mutable data remains under `%LOCALAPPDATA%\OpenVINOWindowsLLM`. Portable mutable data remains under the extracted directory's `data` folder. Review `docs/UPGRADE_ROLLBACK.md` before downgrading.

## Known issues

Real Windows installer upgrade, rollback, Authenticode, CPU, GPU, and NPU paths remain unverified until the corresponding Windows procedures are executed and retained.
