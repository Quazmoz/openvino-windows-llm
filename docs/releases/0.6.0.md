# OpenVINO Windows LLM 0.6.0

## Summary

This release qualifies a modern OpenVINO GenAI dependency stack (OpenVINO / OpenVINO GenAI / OpenVINO Tokenizers `2026.2.1.0`, Optimum Intel `2.0.0`, NNCF `3.2.0`, Transformers `5.0.0`, PyTorch `2.12.0`) and records the first real Intel CPU, GPU, and NPU certification evidence on a physical Intel Core Ultra 9 185H laptop. Two repository defects exposed by the upgrade are fixed, and continuous integration formatting is stabilized.

## New features

- No new end-user API or UI features. This is a dependency-qualification and reliability release.

## Reliability fixes

- **Model conversion no longer crashes on progress glyphs.** Transformers 5.x renders a weight-loading progress bar containing block-drawing characters (U+258F). The converter forwards those UTF-8 lines and printed them to a stdout that falls back to the legacy Windows code page (cp1252) when the server captures it through a pipe, which raised `UnicodeEncodeError` and aborted an otherwise-successful export. The converter now forces UTF-8 stdio, and the server launches the converter subprocess with an explicit UTF-8 environment.
- **Failed model-library imports are now a true no-op on disk.** A failed definition or converted-model import restored the user index by re-serializing it, stamping a fresh `updated_at` instead of restoring the prior bytes. Rollback now restores an exact byte snapshot.

## Continuous integration

- Pinned `ruff==0.16.0` across the dev extra, `requirements-dev.txt`, and `requirements/release.txt` so CI and the release build share one deterministic formatter. The previous configuration floated Ruff (`ruff>=0.6.0`), which had begun resolving a newer Ruff whose formatter reformatted an assert layout and broke `ruff format --check` on `main`; 0.6.0 pins Ruff `0.16.0` so CI and the release build resolve the same formatter.

## Security changes

- No security-relevant changes. Release output scanning, post-sign verification, sanitized progress/diagnostics, and privacy-preserving update checks are unchanged. The diagnostics ZIP was re-inspected on real hardware and contains no usernames, hostnames, tokens, keys, or full private paths.

## API compatibility changes

Chat Completions, Responses, embeddings, streaming, stop/seed/token-limit/context handling, tool-call requests, structured-output requests, and model lifecycle contracts are unchanged and validated with the mock contract suite and against real hardware.

## Model or device behavior changes

- OpenVINO GenAI streaming, `GenerationConfig` (stop strings, seed, sampling, structured output), `LLMPipeline`, `VLMPipeline`, `TextEmbeddingPipeline`, and direct-NPU plugin configuration (`MAX_PROMPT_LEN` / `MIN_RESPONSE_LEN`) all verified against the `2026.2.1.0` API surface.
- FP16 IR produced by earlier OpenVINO releases loads and generates on the `2026.2.1.0` runtime; fresh conversion with the new Optimum/Transformers/NNCF produces loadable artifacts.

## Packaging changes

- Canonical application version advanced to 0.6.0 in `app/version.py`; installer, portable ZIP, checksum, manifest, release-note, and third-party-license filenames follow the deterministic 0.6.0 naming.
- `requirements/release.txt` updated to the qualified, exactly pinned stack. Built from the isolated pinned release environment with a resolved dependency inventory.

## Upgrade notes

Installed mutable data remains under `%LOCALAPPDATA%\OpenVINOWindowsLLM`. Portable mutable data remains under the extracted directory's `data` folder. Review `docs/UPGRADE_ROLLBACK.md` before downgrading.

## Hardware validation performed

On Windows 11 build 26200, Intel Core Ultra 9 185H, Intel Arc iGPU (adapter driver 32.0.101.8425), Intel AI Boost NPU (driver version not captured in the retained reports), with OpenVINO `2026.2.1` / GenAI `2026.2.1.0`, mock mode disabled, requested vs. actual device recorded independently:

- `tinyllama-1.1b-chat-fp16` — CPU, GPU, NPU, and AUTO all PASS (full contract: load, chat, Responses, streaming, early cancel + recovery, stop/seed/limits, tool + structured requests, metrics, unload/reload, benchmark).
- `smollm2-135m-fp16` — CPU, GPU, NPU, and AUTO all PASS (tool/strict-JSON reported as a model-dependent WARN, not a failure).
- `qwen2.5-1.5b-fp16` — CPU, GPU, NPU, and AUTO all PASS.
- `bge-small-en-v1.5` — `/v1/embeddings` PASS on CPU (384 dimensions).
- Converted-artifact reuse (`-SkipConversion`) PASS; fresh conversion round-trip verified.

See `docs/COMPATIBILITY_MATRIX.md` for the per-device rows.

## Mock validation performed

Ruff lint + format, pytest, and the packaged installed/portable mock smoke tests are executed by the release build on Windows before publication.

## Unverified paths

- Authenticode signing and `signtool verify /pa /all` (no signing environment in this qualification).
- `qwen2.5-3b-fp16` quality lane (not converted/certified in this cycle).
- INT4/INT8 precisions (only FP16 certified).
- Long-context depth beyond the short contract prompts.
